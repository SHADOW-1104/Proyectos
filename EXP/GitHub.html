Comandos más usados en Git y GitHub

  • git init: Inicializa un nuevo repositorio Git.
    Ejemplo: git init
  • git clone [url]: Clona un repositorio existente desde una URL.
    Ejemplo: git clone https://github.com/usuario/repositorio.git
  • git status: Muestra el estado de los cambios en el repositorio.
    Ejemplo: git status
  • git add [archivo]: Añade un archivo al área de preparación (staging).
    Ejemplo: git add archivo.txt
  • git commit -m "[mensaje]": Realiza un commit de los cambios en el área de preparación con un mensaje descriptivo.
    Ejemplo: git commit -m "Agregar archivo de configuración"
  • git push: Envía los commits locales al repositorio remoto.
    Ejemplo: git push origin main
  • git pull: Descarga y fusiona los cambios del repositorio remoto al local.
    Ejemplo: git pull origin main
  • git branch: Muestra una lista de las ramas locales.
    Ejemplo: git branch
  • git checkout [rama]: Cambia a una rama específica.
    Ejemplo: git checkout develop
  • git merge [rama]: Fusiona una rama con la rama actual.
    Ejemplo: git merge feature-branch
  • git log: Muestra el historial de commits.
    Ejemplo: git log
  • git remote -v: Muestra las URLs de los repositorios remotos.
    Ejemplo: git remote -v
  • git fetch: Descarga objetos y referencias desde otro repositorio.
    Ejemplo: git fetch origin
  • git diff: Muestra las diferencias entre los archivos modificados pero no aún confirmados.
    Ejemplo: git diff
  • git reset [archivo]: Saca un archivo del área de preparación (staging).
    Ejemplo: git reset archivo.txt
  • git rm [archivo]: Elimina un archivo del repositorio y del sistema de archivos.
    Ejemplo: git rm archivo.txt
  • git stash: Almacena temporalmente los cambios no confirmados.
    Ejemplo: git stash
  • git stash pop: Aplica los cambios almacenados en el stash y los elimina de la lista de stash.
    Ejemplo: git stash pop
  • git rebase [rama]: Reaplica commits en otra base.
    Ejemplo: git rebase main
  • git tag [etiqueta]: Crea una nueva etiqueta.
    Ejemplo: git tag v1.0.0
  • git show [objeto]: Muestra diversos tipos de objetos (commits, etiquetas, etc.).
    Ejemplo: git show v1.0.0
